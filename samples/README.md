# Samples

End-to-end decompilation examples for the ESP32-P4 Ghidra plugin.
Each sample contains the original firmware source, compiled ELF, and the
full pipeline output so you can inspect decompilation quality directly.

## Structure

```
samples/
└── <sample_name>/
    ├── README.md          # accuracy notes + line-count comparison
    ├── input/
    │   ├── *.c            # original C source
    │   ├── *.ld           # linker script
    │   └── *.elf          # compiled firmware binary
    └── output/            # generated by RunFullPipeline.sh
        ├── firmware_decompiled.c   # main decompiled output
        ├── firmware.h              # types + prototypes
        ├── globals.h               # extern declarations
        ├── firmware.dot            # call graph (Graphviz)
        ├── MODULES.md              # module table
        ├── semantic_hints.json     # pattern detection results
        └── pipeline.log            # full analyzeHeadless log
```

## Adding a new sample

1. Place source and ELF in `samples/<name>/input/`
2. Run the pipeline:
   ```bash
   bash tools/RunFullPipeline.sh samples/<name>/input/<name>.elf \
     --out samples/<name>/output
   ```
3. Add a `README.md` with accuracy notes (line counts, variable naming, control flow fidelity)
4. Commit both `input/` and `output/` so the comparison is self-contained in the repo

## Comparing decompiled output to original

```bash
# Side-by-side diff (ignoring Ghidra header comments)
diff <(grep -v '^/[/*]' samples/hello_world/input/hello.c) \
     <(grep -v '^/[/*]' samples/hello_world/output/firmware_decompiled.c)
```
